# Claude Instructions

## About Me
I'm a product designer. My work involves:
- Building HTML mockups and prototypes
- Designing flows and wireframes (Figma, FigJam)
- Website redesigns
- SEO audits and reports

## Code Style
- Prefer functional programming patterns (avoid classes when possible)
- Keep HTML/CSS clean and semantic
- Minimal comments — only where logic isn't obvious
- Prioritize readability over cleverness

## Git Workflow
- Auto-commit after completing tasks — no need to ask first
- Write clear, descriptive commit messages so I can easily review history
- Commit logically grouped changes (don't batch unrelated work into one commit)
- Never force-push or use destructive git operations without asking

## General Preferences
- No strong preference on package manager — use whatever fits the project
- Keep solutions simple and avoid over-engineering
- For HTML mockups, prefer vanilla HTML/CSS/JS unless a framework is clearly needed
- When building prototypes, prioritize visual accuracy over code perfection
